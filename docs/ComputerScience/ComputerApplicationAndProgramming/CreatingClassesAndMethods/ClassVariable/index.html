1. What is a Class?

Think of a class as a blueprint or a template. Just like an architect designs a blueprint to create many houses, in Java, a class defines how objects should look and behave.

For example, imagine you want to create a program that handles information about Cars. You can create a class called Car that describes the attributes (like color, model, speed) and behaviors (methods) like accelerate or brake.

2. How to Define a Class in Java

class Car {
  // Attributes (also called instance variables)
  String color;
  String model;
  int speed;

  // Method to display car info
  void displayInfo() {
    System.out.println("Model: " + model + ", Color: " + color + ", Speed: " + speed + " km/h");
  }
}

Here, Car is the class name. The variables color, model, and speed hold the state of each car object.

3. What is a Class Variable?

A class variable (also called static variable) is shared across all objects created from that class. Unlike instance variables which are unique per object, class variables are common to every instance.

For example, if you want to count how many cars have been created, a class variable can be used:

class Car {
  String model;
  static int numberOfCars = 0;

  Car(String m) {
    model = m;
    numberOfCars++;  // Increase count whenever a new Car is created
  }
}

4. Real World Example: Student Class

Let's make it practical. Suppose we want to track students in a school. Each student has a name and an ID. The school has a total number of students, which is the same for all student objects.

class Student {
  String name;
  int id;
  static int totalStudents = 0;

  Student(String name, int id) {
    this.name = name;
    this.id = id;
    totalStudents++;
  }

  void displayStudent() {
    System.out.println("Name: " + name + ", ID: " + id);
  }

  static void displayTotalStudents() {
    System.out.println("Total Students: " + totalStudents);
  }
}

6. Summary

  • Class: A blueprint for creating objects.
  • Instance Variables: Attributes unique to each object.
  • Class Variables (Static): Shared among all instances of the class.
  • Methods: Functions inside a class that define behavior.

7. Practical Applications

Classes and class variables are everywhere in programming. Here are some examples:

  • Bank Account: Each account object has a balance (instance variable), but the bank's total number of accounts is a class variable.
  • Game Development: Each player has health points and position (instance variables), while the total number of players currently online is a class variable.
  • Library System: Each book has title and author, but the total number of books borrowed can be a class variable.
