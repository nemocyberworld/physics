Java Programming: Overriding Methods

Method Overriding allows a subclass to provide a specific implementation of a method that is already defined in its superclass. It's a key concept in Java's object-oriented design, enabling polymorphism and dynamic method dispatch.

1. What is Method Overriding?

When a method in a subclass has the same name, return type, and parameters as a method in its superclass, the subclass version overrides the superclass method.

class Animal {
  void makeSound() {
    System.out.println("Some generic animal sound");
  }
}

class Dog extends Animal {
  @Override
  void makeSound() {
    System.out.println("Woof Woof");
  }
}

2. Real-World Example: Vehicle

class Vehicle {
  void start() {
    System.out.println("Vehicle is starting");
  }
}

class Car extends Vehicle {
  @Override
  void start() {
    System.out.println("Car is starting with key ignition");
  }
}

4. Rules of Overriding

  • The method must have the same name, return type, and parameters.
  • The subclass method cannot have a more restrictive access modifier.
  • Only inherited methods can be overridden (not static or constructors).

5. Practical Applications

  • UI Components: Override default render or click methods.
  • Games: Override movement or attack behavior for different characters.
  • Payment Gateways: Override a generic `processPayment()` method for credit cards, PayPal, etc.
