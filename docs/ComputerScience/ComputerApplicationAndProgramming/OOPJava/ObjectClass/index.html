Java OOP: Objects and Classes

1. What is Object-Oriented Programming?

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects". These objects can contain data (fields) and code (methods). Java is a pure object-oriented language, which means everything revolves around the use of objects and classes.

2. Understanding Classes

A class is like a blueprint or template. It defines the structure and behavior (data and methods) that the objects created from the class will have.

class Car {
  String color;
  int speed;

  void drive() {
    System.out.println("The car is driving.");
  }
}

3. Understanding Objects

An object is an instance of a class. You create objects using the new keyword:

Car myCar = new Car();
myCar.color = "Red";
myCar.speed = 100;
myCar.drive();

4. Real-World Examples

  • Bank Account: A class can represent a bank account with attributes like balance, and methods like deposit() and withdraw().
  • Student: A Student class may include name, roll number, and marks with methods like study() or giveExam().
  • Smartphone: With properties like brand, model, and memory, and methods like makeCall() or takePhoto().

5. Fun Fact!

Did you know? Java was originally developed for interactive television, but it was too advanced for the digital cable television industry at the time. The language was initially called Oak!

6. Practical Applications

Understanding objects and classes is essential for building:

  • Mobile applications (e.g., Android apps)
  • Web applications (e.g., banking sites)
  • Games (object-oriented engines use classes for characters, weapons, etc.)
  • Enterprise software (billing, customer management)

7. Creating Multiple Objects

Car car1 = new Car();
Car car2 = new Car();
car1.color = "Blue";
car2.color = "Green";
System.out.println(car1.color); // Blue
System.out.println(car2.color); // Green

8. Summary

  • Class: Blueprint
  • Object: Instance of class
  • Method: Function inside a class
  • Field: Variable inside a class
