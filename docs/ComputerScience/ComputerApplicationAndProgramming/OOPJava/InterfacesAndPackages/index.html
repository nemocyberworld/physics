Java OOP: Interfaces and Packages

1. What is an Interface?

An interface in Java is a contract that a class agrees to fulfill. It is a collection of abstract methods (methods without bodies). A class implements an interface to provide concrete behavior.

2. Syntax of an Interface

interface Animal {
  void makeSound();
}

class Dog implements Animal {
  public void makeSound() {
    System.out.println("Woof woof!");
  }
}

3. Real-World Example

Think of an interface as a remote control. Different devices (TV, Fan, AC) implement the same interface to respond to the powerOn() button.

5. Why Use Interfaces?

  • Multiple Inheritance: Java allows a class to implement multiple interfaces.
  • Abstraction: Hides implementation and exposes behavior.
  • Loose Coupling: Helps in scalable and testable architecture.

6. Practical Examples of Interfaces

  • Comparable Interface: For sorting objects.
  • Runnable Interface: For creating threads.
  • In game design, different weapons or players might implement a Usable interface.

7. Fun Fact About Interfaces!

Java 8 introduced default methods in interfaces, allowing methods with bodies — bridging the gap between interfaces and abstract classes!

8. What are Packages?

A package is a namespace for organizing classes and interfaces. Think of it as a folder structure for your code that prevents name conflicts and promotes modularization.

9. Declaring a Package

package animals;

public class Dog {
  public void bark() {
    System.out.println("Woof!");
  }
}

10. Importing a Package

import animals.Dog;

public class Main {
  public static void main(String[] args) {
    Dog d = new Dog();
    d.bark();
  }
}

11. Why Use Packages?

  • Organize code better.
  • Prevent class name conflicts.
  • Control access with visibility modifiers.

13. Summary

  • Interfaces define contracts for behavior.
  • Packages organize your Java code effectively.
  • Both support scalable and modular application development.
