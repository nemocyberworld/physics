Inheritance – Extending Classes in Object-Oriented C++

Inheritance is a fundamental concept in Object-Oriented Programming (OOP) that allows one class to inherit properties and behaviors (data members and functions) from another class. This promotes code reuse, establishes relationships between classes, and supports polymorphism.

What is Inheritance?

In C++, inheritance lets you create a new class (called the derived class or child) based on an existing class (called the base class or parent). The derived class inherits attributes and methods from the base class and can also add its own unique features.

Basic Syntax of Inheritance in C++

class BaseClass {
  // base class members
};

class DerivedClass : access_specifier BaseClass {
  // derived class members
};

The access_specifier is usually public, protected, or private, with public being most common.

Simple Example: Animals and Dogs

Let's say we have a base class Animal that defines basic animal properties and behaviors. We want to create a derived class Dog that inherits from Animal but also has dog-specific behavior. 

class Animal {
public:
  void eat() {
    std::cout << "This animal eats food." << std::endl;
  }
};

class Dog : public Animal {
public:
  void bark() {
    std::cout << "The dog barks: Woof!" << std::endl;
  }
};

Here, Dog inherits the eat() method from Animal and also has its own method bark().

Real World Example: Vehicles

Consider a base class Vehicle and derived classes Car and Bike. All vehicles have common properties like speed and capacity, but each type can have unique features. 

class Vehicle {
public:
  int speed;
  int capacity;

  void start() {
    std::cout << "Vehicle started." << std::endl;
  }
};

class Car : public Vehicle {
public:
  int numDoors;

  void openTrunk() {
    std::cout << "Trunk is open." << std::endl;
  }
};

class Bike : public Vehicle {
public:
  bool hasCarrier;

  void ringBell() {
    std::cout << "Ring ring!" << std::endl;
  }
};

Access Specifiers and Inheritance

When inheriting, access specifiers (public, protected, private) control visibility of base class members:

  • public inheritance: public and protected members stay public/protected in derived class.
  • protected inheritance: public and protected members become protected in derived class.
  • private inheritance: public and protected members become private in derived class.

Public inheritance is most common and models "is-a" relationships (e.g., a Dog is an Animal).

Types of Inheritance in C++

  • Single inheritance (one base, one derived class)
  • Multiple inheritance (a derived class inherits from multiple base classes)
  • Multilevel inheritance (chain of inheritance like grandparent → parent → child)
  • Hierarchical inheritance (multiple derived classes from one base class)

Fun Fact!

The concept of inheritance in programming is inspired by real-world biological inheritance, where offspring inherit traits from their parents.

Practical Applications of Inheritance

  • GUI Frameworks: Base classes define common controls; derived classes extend with specific features (e.g., Button, TextBox).
  • Game Development: Characters inherit common properties from a base class, with specialized subclasses for enemies, players, and NPCs.
  • Automobile Software: Base vehicle class extended by specific car, truck, or motorcycle classes with additional features.
  • Banking Systems: Base Account class extended by SavingsAccount, CheckingAccount classes with specific rules.

Summary

Inheritance is a powerful mechanism to create a new class from an existing one, promoting code reuse and logical structure. Understanding inheritance is key to mastering Object-Oriented Programming in C++.
