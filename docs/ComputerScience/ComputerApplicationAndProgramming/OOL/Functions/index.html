Functions in Object-Oriented C++

Welcome to this interactive lesson on Functions in Object-Oriented Programming using C++. Functions are fundamental building blocks in programming—they help organize code into reusable blocks that perform specific tasks.

What is a Function?

A function is a block of code that performs a particular task. It can take input parameters, execute code, and optionally return a value. In Object-Oriented Programming (OOP), functions are often called methods when they are defined inside a class.

Function Syntax in C++

return_type function_name(parameter_list) {
    // body of the function
    // code to be executed
}

Example:

int add(int a, int b) {
    return a + b;
}

Functions in Classes (Methods)

In OOP, functions inside classes are called methods. They operate on data contained in the class and help define the behavior of objects. 

class Car {
  public:
    void startEngine() {
      std::cout << "Engine started!" << std::endl;
    }
};

Real World Example: Bank Account

Imagine you are building a banking application. You need to create an account, deposit money, withdraw money, and check the balance.

class BankAccount {
  private:
    double balance;

  public:
    BankAccount() { balance = 0; }

    void deposit(double amount) {
      balance += amount;
      std::cout << "Deposited: $" << amount << std::endl;
    }

    void withdraw(double amount) {
      if(amount <= balance) {
        balance -= amount;
        std::cout << "Withdrawn: $" << amount << std::endl;
      } else {
        std::cout << "Insufficient funds!" << std::endl;
      }
    }

    double getBalance() {
      return balance;
    }
};

Why Use Functions?

  • Code Reusability: Write once, use multiple times.
  • Organization: Break large programs into manageable pieces.
  • Abstraction: Hide complex details, show only what’s necessary.
  • Maintainability: Easier to update and fix bugs.

Advanced: Function Overloading in C++

Function overloading means having multiple functions with the same name but different parameter lists. The compiler decides which one to call based on the arguments. 

int add(int a, int b) {
  return a + b;
}

double add(double a, double b) {
  return a + b;
}

Fun Fact!

The concept of functions dates back to ancient mathematics, but the term "function" was formally introduced by the mathematician Gottfried Wilhelm Leibniz in the 17th century.

Practical Applications of Functions in OOP

  • Game Development: Functions handle player actions, score calculations, and game logic.
  • Web Browsers: Functions render pages, manage user input, and handle network requests.
  • Banking Software: Functions manage transactions, interest calculations, and reporting.
  • Robotics: Functions control sensors, motors, and decision-making algorithms.

Summary

Functions are the heart of programming, especially in Object-Oriented Languages like C++. They enable encapsulation, reuse, and modularity. Understanding functions will give you a strong foundation to build complex and efficient programs.
