C++ as an Object-Oriented Programming Language

Welcome to this comprehensive lesson on Object-Oriented Programming (OOP) in C++. Whether you're brand new or just brushing up, you'll find detailed explanations, real-world examples, practical applications, and fun interactive bits to help solidify your understanding.

What is Object-Oriented Programming?

Object-Oriented Programming is a programming paradigm based on the concept of "objects". These objects can contain data, in the form of fields (often called attributes or properties), and code, in the form of procedures (often called methods).

C++ is a powerful language that supports OOP features such as classes, inheritance, encapsulation, and polymorphism.

Key Concepts of OOP in C++

1. Classes and Objects

A class is like a blueprint for creating objects. An object is an instance of a class.

class Car {
  public:
    string brand;
    string model;
    int year;
  
    void honk() {
      cout << "Beep beep!" << endl;
    }
};

Here, Car is a class with properties like brand, model, and year, and a method called honk().

Real-world example:

Think of a Car blueprint. Each car you buy (a Honda Civic, a Toyota Corolla) is an object created from this blueprint. Each car has attributes (color, brand, model year) and behaviors (drive, honk, stop).

Car myCar;                 // Creating an object of type Car
myCar.brand = "Toyota";
myCar.model = "Corolla";
myCar.year = 2020;
myCar.honk();              // Output: Beep beep!

2. Encapsulation

Encapsulation means bundling data (variables) and methods (functions) that operate on the data into a single unit or class. It also hides the internal state of the object from the outside world â€” often done using private access specifier.

class BankAccount {
  private:
    double balance;
  public:
    BankAccount(double initialBalance) {
      balance = initialBalance;
    }
    void deposit(double amount) {
      balance += amount;
    }
    void withdraw(double amount) {
      if (amount <= balance) {
        balance -= amount;
      } else {
        cout << "Insufficient funds." << endl;
      }
    }
    double getBalance() {
      return balance;
    }
};

Real-world application:

Bank accounts are classic examples of encapsulation. You cannot directly modify the balance without going through methods like deposit or withdraw. This protects the integrity of your money.

BankAccount myAccount(1000);
myAccount.deposit(500);
myAccount.withdraw(200);
cout << "Balance: " << myAccount.getBalance() << endl;  // Balance: 1300

3. Inheritance

Inheritance allows a class to inherit properties and methods from another class. This promotes code reuse and establishes a relationship between classes.

class Vehicle {
  public:
    void start() {
      cout << "Vehicle started" << endl;
    }
};

class Car : public Vehicle {
  public:
    void honk() {
      cout << "Car honking" << endl;
    }
};

Real-world example:

A Car is a kind of Vehicle. It inherits the ability to start from Vehicle, but also has its own behavior, like honking.

Car myCar;
myCar.start();  // Vehicle started
myCar.honk();   // Car honking

4. Polymorphism

Polymorphism allows methods to do different things based on the object it is acting on. This can be achieved using function overriding with virtual functions.

class Animal {
  public:
    virtual void sound() {
      cout << "Some generic animal sound" << endl;
    }
};

class Dog : public Animal {
  public:
    void sound() override {
      cout << "Bark" << endl;
    }
};

class Cat : public Animal {
  public:
    void sound() override {
      cout << "Meow" << endl;
    }
};

Practical use:

You can write code that works on Animal pointers, but the actual sound() function that gets called depends on the specific type of animal.

Animal* a1 = new Dog();
Animal* a2 = new Cat();

a1->sound();   // Bark
a2->sound();   // Meow

Why Use Object-Oriented Programming in C++?

  • Modularity: Code is organized into classes making it easier to maintain.
  • Reusability: Inheritance and polymorphism reduce code duplication.
  • Flexibility: Polymorphism allows flexible and easily extendable code.
  • Real-World Modeling: Closely models real-world entities.

Practical Applications of OOP in C++

  • Game Development: Objects represent characters, enemies, and items with behaviors.
  • GUI Applications: Windows, buttons, and menus are objects with events and properties.
  • Simulations: Simulate real-world systems like traffic, weather, or physics.
  • Banking Systems: Secure encapsulated transactions.
  • Embedded Systems: Hardware components can be modeled as objects.
